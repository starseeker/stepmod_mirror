Reference: Elemental_topology_arm.Connected_edge_set
Description:
A Connected_edge_set is a type of Detailed_topological_model_element that is a set of edges arcwise connected via their common vertices.
Reference: Elemental_topology_arm.Connected_edge_set.connected_edges
Description:
a set of Edges arcwise connected at common Vertexs. The domains of the connected_edges shall not intersect.
Reference: Elemental_topology_arm.Connected_face_set
Description:
A Connected_face_set is a type of Detailed_topological_model_element that is a set of faces such that the domain of the faces together with their bounding edges and vertices is connected.
Reference: Elemental_topology_arm.Connected_face_set.connected_faces
Description:
a set of Faces arcwise connected along common Edges or Vertexs. The domains of the connected_faces shall not overlap.
Reference: Elemental_topology_arm.Connected_face_sub_set
Description:
A Connected_face_sub_set is a type of Connected_face_set.
Reference: Elemental_topology_arm.Connected_face_sub_set.parent_face_set
Description:
specifies a role of the Connected_face_set for the Connected_face_sub_set.
Reference: Elemental_topology_arm.Detailed_topological_model_element
Description:
A Detailed_topological_model_element is a type of Representation_item that represents a single element of the topology, or connectivity, of the entities that make up the representation of an object. A Detailed_topological_model_element is either a Connected_edge_set or a Connected_face_set, or an Edge, or a Face, or a Face_bound, or a Vertex, or a Loop, or a Loop and a Path, or a Path.
Reference: Elemental_topology_arm.Detailed_topological_model_element.permanent_id
Description:
an identifier included to provide identification that is invariant over the life cycle of the product model.

NOTE    An application protocol may wish to enforce uniqueness constraints on the value of this attribute to ensure there is no sharing of the identifier among Detailed_topological_model_element in the context of a product model version.

Reference: Elemental_topology_arm.Detailed_topological_model_element.permanent_aggregate_id
Description:
an identifier included to provide identification that is invariant over the life cycle of the product model. This attribute is designed to support the use case where more than one member of Detailed_topological_model_element may share the same identifier within the context of a specific product model version.
Reference: Elemental_topology_arm.Edge
Description:
An Edge is a type of Detailed_topological_model_element that is an oriented bounded one dimensional manifold connecting two vertices. The topological sense of the edge is from the edge_start to the edge_end.
Reference: Elemental_topology_arm.Edge.edge_start
Description:
the Vertex defining the start point of the edge.
Reference: Elemental_topology_arm.Edge.edge_end
Description:
the Vertex defining the end point of the edge; this may coincide with the start point if the edge forms a closed loop.
Reference: Elemental_topology_arm.Edge_loop
Description:
An Edge_loop is a type of Loop with a non-zero extent. It is also a type of Path with the property that the start and end vertices coincide.
Reference: Elemental_topology_arm.Edge_loop.ne
Description:
the number of Edges defining the path of the loop.
Reference: Elemental_topology_arm.Edge_loop.wr:WR1
Description:
The edge_start of the first edge shall be the same as the edge_end of the last edge. This ensures that the path is closed to form a loop.
Reference: Elemental_topology_arm.Face
Description:
A Face is a type of Detailed_topological_model_element that is a topological entity of dimensionality 2 corresponding to the intuitive notion of a piece of surface bounded by loops. Its domain, if present, is an oriented, connected, finite 2-manifold. A face domain shall not have handles, but it may have holes, each hole bounded by a loop.

NOTE    The full semantics of this entity is defined in the corresponding MIM object from ISO 10303-42; this has additional constraints in the form of informal propositions.

Description:
the set of Face_bounds which define all the boundaries of the face.
Reference: Elemental_topology_arm.Face.wr:WR1

At most, one of the bounds shall be of type Face_outer_bound.
Reference: Elemental_topology_arm.Face_bound
Description:
A Face_bound is a type of Detailed_topological_model_element that is the usage of a loop for bounding a Face.
Reference: Elemental_topology_arm.Face_bound.bound

the Loop which will be used as a face boundary.
Reference: Elemental_topology_arm.Face_bound.orientation
Description:
an indicator of whether (TRUE), or not (FALSE) the Loop has the same sense when used to bound the face as when first defined. If orientation is FALSE, the senses of all its component oriented edges are implicitly reversed when used in the Face.
Reference: Elemental_topology_arm.Face_outer_bound
Description:
A Face_outer_bound is a type of Face_bound which carries the additional semantics of defining an outer boundary on the face. A Face_outer_bound shall separate the interior of the face from the exterior and shall enclose the interior domain of the face. No more than one boundary of a face shall be of this type.
Reference: Elemental_topology_arm.Loop
Description:
A Loop is a type of Detailed_topological_model_element constructed from a single Vertex, or by stringing together connected edges beginning and ending at the same Vertex. A Loop has dimensionality 0 or 1. A Loop may be either a Vertex_loop or an Edge_loop.
Reference: Elemental_topology_arm.Oriented_edge
Description:
An Oriented_edge is a type of Edge constructed from another edge which contains a BOOLEAN orientation flag to indicate whether or not the orientation of the constructed edge agrees with the orientation of the original edge. Except for possible re-orientation, the Oriented_edge is equivalent to the original edge.
Reference: Elemental_topology_arm.Oriented_edge.edge_definition
Description:
the Edge defining the domain of the Oriented_edge
Reference: Elemental_topology_arm.Oriented_edge.orientation
Description:
a BOOLEAN flag. If TRUE, the topological orientation as used coincides with the orientation, from start vertex to end vertex, of the edge_definition, if FALSE the vertices are reversed in order.
Reference: Elemental_topology_arm.Oriented_edge.edge_start
Description:
the start vertex of the Oriented_edge. This is derived from the vertices of the edge_definition after taking account of the orientation.
Reference: Elemental_topology_arm.Oriented_edge.edge_end
Description:
the end vertex of the Oriented_edge. This is derived from the vertices of the edge_definition after taking account of the orientation.
Reference: Elemental_topology_arm.Oriented_edge.wr:WR1
Description:

Reference: Elemental_topology_arm.Path
Description:
A Path is a type of Detailed_topological_model_element consisting of an ordered collection of Oriented_edges, such that the start vertex of each edge coincides with the end vertex of its predecessor. The path is ordered from the edge_start of its first Oriented_edge to the edge_end of its last edge. The BOOLEAN value orientation in the Oriented_edge indicates whether the Edge direction agrees with the direction of the Path (TRUE) or is in the opposite direction (FALSE).

An individual edge can only be referenced once by an individual path. An edge can be referenced by multiple paths. An edge can exist independently of a path.

Reference: Elemental_topology_arm.Path.edge_list
Description:
the list of unique Oriented_edge entities which are concatenated together to form this Path.
Reference: Elemental_topology_arm.Path.wr:WR1
Description:
The end vertex of each Oriented_edge shall be the same as the start vertex of its successor. This is checked by the function path_aligned.

Reference: Elemental_topology_arm.path_aligned
Description:
