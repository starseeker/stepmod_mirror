Introduction

The purpose is to specify numbers, ranges of numbers, and their units of measure

1 Scope

This part of ISO 10303 specifies the application module for Value with unit. The following are within scope of this part of ISO 10303:

  • values that are single numbers;
  • values that are all numbers in a range;
  • units.

The following are outside the scope of this part of ISO 10303:

  • all things which are not involved in the specification of the concepts of value with unit.

4 Information requirements

This clause specifies the information requirements for Value with unit application module.

The information requirements are specified as a set of units of functionality and application objects. The information requirements are defined using the terminology of the subject area of this application module.

4.1 Units of functionality

 This subclause specifies the units of functionality (UoF) for this part ISO 10303 as well as any support elements needed for the application module definition. This part of ISO 10303 specifies the following units of functionality and application objects.
  • Value_With_Unit.

This part of ISO 10303 uses no other units of functionality.

The units of functionality and a description of the functions that each UoF supports are given below.

4.1.1 Value_With_Unit

This UoF supports the specification of numerical values with units of measure.

The following application objects are defined in the Value_With_Unit UoF:

  • Numerical_value;
  • Unit;
  • Value_limit;
  • Value_range;
  • Value_with_unit.

4.2 ARM entity definitions

This subclause specifies the application entities for this module. Each application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The application entities and their definitions are given below.

4.2.1 Numerical_value

A Numerical_value specifies a quantity expressed with a single numerical value and a unit

EXPRESS specification:

*)
ENTITY Numerical_value
  SUBTYPE OF (Value_with_unit) ;
   value_component : UNDEFINED_OBJECT;
END_ENTITY;
(*

Attribute definitions:

value_component: the quantity of the Numerical_value.

4.2.2 Unit

A Unit specifies a quantity chosen as a standard in terms of which other quantities may be expressed.

EXPRESS specification:

*)
ENTITY Unit;
   unit_name : STRING;
END_ENTITY;
(*

Attribute definitions:

unit_name: the term representing the kind of unit. EXAMPLE gram, litre, or volt.

4.2.3 Value_limit

A Value_limit specifies a qualified numerical value representing either the lower limit or the upper limit of a particular physical characteristic. EXAMPLE -- ’30.5 max’ and ’5 min’ are examples for a Value_limit.

EXPRESS specification:

*)
ENTITY Value_limit
  SUBTYPE OF (Value_with_unit) ;
   limit_qualifier : STRING;
   limit : STRING;
END_ENTITY;
(*

Attribute definitions:

limit_qualifier: The limit_qualifier specifies the kind of limit. The following values shall be used: ’maximum’: The specified limit is an upper limit; ’minimum’: The specified limit is a lower limit.
limit: the value of the limit.

4.2.4 Value_range

A Value_range specifies a pair of numerical values representing the range in which the value shall lie.

EXPRESS specification:

*)
ENTITY Value_range
  SUBTYPE OF (Value_with_unit) ;
   upper_limit : STRING;
   lower_limit : STRING;
END_ENTITY;
(*

Attribute definitions:

upper_limit: the maximum value of the Value_range

4.2.5 Value_with_unit

A Value_with_unit specifies either a single numerical measure, or a range of numerical measures with upper, lower, or upper and lower bounds.

EXPRESS specification:

*)
ENTITY Value_with_unit
  ABSTRACT SUPERTYPE   OF (ONEOF(Numerical_value, Value_range, Value_limit)) ) ;

END_ENTITY;

Attribute definitions:

unit_component: the unit in which the Value_with_unit is expressed.
significant_digits: the number of decimal digits that are relevant for the use of the Value_with_unit. If present, the numerical measure or range may be specified using more digits than the significant digits but shall not be specified using fewer digits.

ENTITY value_range
  SUBTYPE OF (compound_representation_item) ;
  WHERE
    WR1: ( 'value_with_unit_mim.'+ 'SET_REPRESENTATION_ITEM' IN TYPEOF ( item_element ) ) AND value_range_wr1 ( item_element ) ;
    WR2value_range_wr2: value_range_wr2 ( item_element ) ;
    WR3value_range_wr3: value_range_wr3 ( item_element ) ;
END_ENTITY;

FUNCTION value_range_wr1 (agg : AGGREGATE OF representation_item) : BOOLEAN; 
  BEGIN
    IF (SIZEOF(agg) = 2) AND ((SIZEOF(QUERY ( i1 <* agg | (
    'value_with_unit_mim.MEASURE_REPRESENTATION_ITEM' IN TYPEOF
    (i1)) ))
    = 2) OR (SIZEOF(QUERY ( i2 <* agg | (
    'value_with_unit_mim.VALUE_REPRESENTATION_ITEM' IN TYPEOF
    (i2)) ))
    = 2)) THEN
      RETURN(TRUE);
    ELSE
      RETURN(FALSE);
    END_IF;
  END;
END_FUNCTION;

FUNCTION value_range_wr2 (agg : AGGREGATE OF representation_item) : BOOLEAN; 
  BEGIN
    IF (SIZEOF(QUERY ( i <* agg | (i.name = 'upper limit')
        )) = 1)
    AND (SIZEOF(QUERY ( i <* agg | (i.name = 'lower limit')
         )) = 1)
     THEN
      RETURN(TRUE);
    ELSE
      RETURN(FALSE);
    END_IF;
  END;
END_FUNCTION;
