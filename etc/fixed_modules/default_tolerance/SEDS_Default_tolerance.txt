Summary/Abstract/Keywords: Multiple mapping syntax errors.

Problem Description: 

Mapping syntax error - "=" is used for other case than select:
set_representation_item[i] = representation_item

It has to be
set_representation_item[i] -&gt; representation_item 

It occurrs in the following mappings:
Tolerance_table_cell.cell_value (to Plus_minus_toleranced_datum)
Tolerance_table_cell.cell_value (to Upper_lower_toleranced_datum)
Tolerance_table_cell.description
Tolerance_table_cell.limit(to Significant_number_of_digits)
Tolerance_table_cell.limit(to Upper_lower_limit)

Conditions Under Which the Issue Was Discovered: Validation with LKSoft mapping tool.

Proposed Solution (Optional):
In the following mappings:
Tolerance_table_cell.cell_value (to Plus_minus_toleranced_datum)
Tolerance_table_cell.cell_value (to Upper_lower_toleranced_datum)
Tolerance_table_cell.description
Tolerance_table_cell.limit(to Significant_number_of_digits)
Tolerance_table_cell.limit(to Upper_lower_limit)

Change
set_representation_item[i] = representation_item

To
set_representation_item[i] -&gt; representation_item
