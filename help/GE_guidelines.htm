Graphical Express additional directives

While in GE

 Eurostep logo 

To eliminate the logo and  incorrect title, cover with a rectangle in VISIO.

or, alternatively  after exporting as XML, crop the *.gif images (see below).

 

Required comments

The best way to get to get static comments into arm.exp or mim.exp , is to go to the schema page and click on main schema, then add them to the description field.

Example: 

  $Id:  $
  N773 - ISO TS - 10303-1042 Work request - EXPRESS MIM

NOTE : in above, any text after the colon and before the second dollar sign was added by CVS - I did not put it in... This is how it is supposed to work.

Remark  delimiters will be added to Express by GE export process.

Exporting from GE

Export EXPRESS and Publish as XML.

Now run ge2module.wsf

Cleanup of files after running ge2module.wsf.

.exp files: 

Remove language identifier if any.

Remove Eurostep comments.

Remove any extra schemas.

Add N number and cvs keyword - note : these can be added to the GE model once instead of in outputs each time you must modfiy.

 Express-G diagrams 

armexp?.gif and mimexp?.gif

If Eurostep logo is still visible, 

  • crop the gif images, ( for example with  Microsoft Photo Editor). Be sure to crop only bottom and right side, retaining original settings for top and left side. This will assure that image maps still work. 

arm.xml and mim.xml

Add  to express element the attribute description.file = "arm_descriptions.xml"   or "mim_descriptions.xml" 

Add selects="YES" to get boilerplate descriptions for selects.

armexpg?.xml or mimexpg?.xml

If there is no entity level diagram for the mim schema (or occasionaly the arm schema) , replace the href in the image map with an href to the mim listing in clause 5. 

EXAMPLE  For Shape_property_assignment_mim: href="../shape_property_assignment/sys/5_mim.xml#mim_express"
