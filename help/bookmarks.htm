All significant document sections in the module repository have been bookmarked (labelled as a named anchor using the HTML <a name="label"> construct). This includes document sections such as the scope, introduction etc as well as all the EXPRESS elements (entities, types etc) of the ARM, MIM and Integrated resource models.

The labels can be referenced from an XML document using the XML elements <express_ref> and <module_ref>. See XML referencing.

The HTML labels are derived from the XML by the XSL style sheets and can be referenced from HTML files external to the repository using standard HTML: <a href="label">.

The HTML labels are described in "HTML bookmarking". Note: these are the HTML labels derived from the XML.

XML referencing

 The EXPRESS entities and the sections of module text can be referenced using the XML elements <express_ref> and <module_ref> respectively.

EXPRESS reference

All EXPRESS constructs in a module ARM or MIM or in an integrated resource are bookmarked (i.e. labelled as a named anchor using the <a name="label"> construct). See section "EXPRESS HTML bookmarks" for details.

Hyper links can be set up to these bookmarked EXPRESS constructs from the XML using the XML element: <express_ref>. This will result in <a href="label"> constructs in the HTML.

The DTD is: 

  <!ELEMENT  express_ref (#PCDATA)>
  <!ATTLIST  express_ref linkend CDATA #REQUIRED>

The <linkend> attribute is the label for the hyper link referencing the EXPRESS construct in a module ARM or MIM or Integrated Resource. It is a structured string with the following format:

<module>:ir_express|mim|mim_express|arm|arm_express:<schema>.<entity|type|function|procedure|constant>.<attribute>|wr:<whererule>|ur:<uniquerule>
where:
<module>
is the name of the module.

ir_express|mim|mim_express|arm|arm_express
indicate that the reference is to the Integrated Resource, ARM or MIM EXPRESS.

ir_express
indicates that the reference is to the Integrated Resource EXPRESS listing.

mim
indicates that the reference is to the MIM EXPRESS described in section 5 of the module documentation.

mim_express
indicates that the reference is to the MIM EXPRESS listing.

arm
indicates that the reference is to the ARM EXPRESS described in section 4 of the module documentation.

arm_express
indicates that the reference is to the ARM EXPRESS listing.

<schema>
is the name of the schema.

<entity|type|function|procedure|constant>
is the name of the entity, type, function, procedure or constant.

<attribute>
is the name of the attribute

<whererule>
is the name of the where rule.

<uniquerule>
is the name of the unique rule.

Example

A reference to the ARM description of the EXPRESS attribute "name", on the entity "Activity", defined in the "work_order_arm" schema, which is the ARM schema for the "work_order" module is expressed as: 

  <express_ref
      linkend="work_order:arm:work_order_arm.Activity.name">
   work_order:arm:work_order_arm.Activity.name
  </express_ref>

Will result in: work_order:arm:work_order_arm.Activity.name being displayed.

The resulting HTML is: 

  <a href="../data/modules/work_order/sys/4_info_reqs.htm#work_order_arm.activity.name">
  work_order:arm:work_order_arm.Activity.name</a>

Example

Module reference

 All sections of module text including figures, tables, notes and examples are bookmarked (i.e. labelled as a named anchor using the <a name="label"> construct). See section "Module document HTML bookmarks" for details.

Hyper links can be set up to these bookmarked module sections from the XML using the XML element: <module_ref>. This will result in <a href="label"> constructs in the HTML.

The DTD is: 

  <!ELEMENT  module_ref (#PCDATA)>
  <!ATTLIST  module_ref linkend CDATA #REQUIRED>

The <linkend> attribute is the label for the hyper link referencing the EXPRESS construct in a module ARM or MIM or Integrated Resource. It is a structured string with the following format:
<module>:<section>:<construct>:<id>
where:
<module>
is the name of the module.
<section>
identifies the section in the module document.

introduction
The introduction section.

1_scope
The scope section.

1_inscope
The in scope section.

1_outscope
The out of scope section.

2_normrefs
The normative references section.

3_definition
The definitions section.

3_abbreviations
The abbreviations section.

4_uof
The Units of functionality section.

4_interfaces
The ARM interfaces.

4_constants
The ARM EXPRESS constants.

4_types
The ARM EXPRESS types.

4_entities
The ARM EXPRESS entities.

4_rules
The ARM EXPRESS rules.

4_functions
The ARM EXPRESS functions.

4_procedures
The ARM EXPRESS procedures.

5_mapping
The ARM to MIM mapping section.

5_express
The ARM to MIM EXPRESS section.

5_interfaces
The MIM EXPRESS intefaces.

5_constants
The MIM EXPRESS constants.

5_types
The MIM EXPRESS types.

5_entities
The MIM EXPRESS entities.

5_rules
The MIM EXPRESS rules.

5_functions
The MIM EXPRESS functions.

5_procedures
The MIM EXPRESS procdures.

f_usage_guide
The usage guide section.

<construct>
is optional and indentifies whether a figure, table, note or example is being referenced,

figure
A figure is being referenced. The identifier for the figure is given by the id construct.

note
A note is being referenced. The identifier for the note is given by the id construct.

example
An example is being referenced. The identifier for the example is given by the id construct.

table
A table is being referenced. The identifier for the table is given by the id construct.

<id>
if a construct is given, then id is the identifier for the construct.

Example

A reference to the inscope section of the work_order module. 

  <module_ref 
       linkend="work_order:1_scope">
      work order scope.
  </module_ref>

Will result in: work order scope.

The resulting HTML is: 

  <href="../data/modules/work_order/sys/1_scope.htm#outscope">
      work order scope.</a>

Example

A reference to figure 1 in introduction to the work_order module. 

  <module_ref 
       linkend="work_order:introduction:figure:1">
      Figure 1 in Work_order introduction
  </module_ref>

Will result in: work order scope.

The resulting HTML is: 

  <href="../data/modules/work_order/sys/introduction.htm#figure:1">
      Figure 1 in Work_order introduction</a>

Figures, notes etc HTML bookmarks

Each figure, table, note and example has an HTML anchor. The format is:
figure|note|example|table:<id>
where <id> is either the number of the figure, note etc or the identifier specified in the XML element:
<figure id="identifier">
<note id="identifier">
<example id="identifier">
<table id="identifier">

EXPRESS HTML bookmarks

Each EXPRESS construct has an HTML anchor. The format is: (all in lower case)
<schema>.<entity|type|function|procedure|constant>.<attribute>|wr:<whererule>|ur:<uniquerule>
where:
<schema>
is the name of the schema.

<entity|type|function|procedure|constant>
is the name of the entity, type, function, procedure or constant.

<attribute>
is the name of the attribute

<whererule>
is the name of the where rule.

<uniquerule>
is the name of the unique rule.

For example, consider the following EXPRESS code: 

SCHEMA management_resources_schema;
ENTITY action_assignment
  ABSTRACT SUPERTYPE;
  assigned_action : action;
DERIVE
  role            : object_role := get_role (SELF);
WHERE
  WR1 : SIZEOF (USEDIN (SELF, 'BASIC_ATTRIBUTE_SCHEMA.' +
                       'ROLE_ASSOCIATION.ITEM_WITH_ROLE')) <= 1;
END_ENTITY; -- action_assignment
END_SCHEMA; -- management_resources_schema

The anchor for the schema management_resources_schema will be:

<a name="management_resources_schema"/>

The anchor for the entity action_assignment will be:

<a name="management_resources_schema.action_assignment"/>

The anchor for the attribute assigned_action will be:

<a name="management_resources_schema.action_assignment.assigned_action"/>

The anchor for the where rule WR1 will be:

<a name="management_resources_schema.action_assignment.wr:wr1"/>
