Introduction

A Bugzilla instance is used to manage and track the issues raised against the modules. The Bugzilla for ISO 10303 is at: http://locke.dcnicn.com/bugzilla/iso10303/.

Getting started with Bugzilla

Bugzilla is web based. In order to use the ISO 10303 bugzilla, you need to be a user. To create an account got to: http://locke.dcnicn.com/bugzilla/iso10303/createaccount.cgi.

ISO 10303 Bugzilla structure

Bugzilla is organized by products. For ISO 10303 the following products have been setup. Issues should be raised against the relevant product.

ISO 10303:
All parts of ISO 10303
Recommended Practices and Guides:
All Recommended Practices and other guidelines on how to use ISO 10303
stepmod environment and utilities:
All xsl, dtd and utilities that make stepmod running

Recording ballot comments

Ballot comments should be recorded as bugzilla issues as follows:

  1. Create a bug which collects all the ballot comments bugs i.e. have them as blockers
  2. Raise a bug for each ballot comment. Use the following fields as follows
    • Summary: start summary with: "Ballot_UK: " "Ballot_US:" etc rather like all SEDS start with SEDS
    • OS: "Ballot Comment"
    • Comment: the ballot comment itself.

Addressing ballot comments

When the ballot has been resolved, the target milestone (release) in which the resolved issue is incorporated should be identified. The milestone is either a Part1000 change request for issues against a module, or new edition of an AP or Integrated Resource. This should reflect the ballot stage. E.g. AP239ed2-DIS

The status of the issue should be set according to the normal bugzilla process. See "Bugzilla states".
