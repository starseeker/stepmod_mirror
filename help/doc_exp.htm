EXPRESS files

Each module contains two EXPRESS files, arm.exp and mim.exp for the ARM and MIM respectively.

An XML presentation of these EXPRESS files is derived and stored in arm.xml and mim.xml and is used to document the EXPRESS.

The documentation for the EXPRESS can be provided in two ways:

  • Included in the XML file through the <description> element,
  • External to the XML file through the description files.

The XML elements that can be used in the description text (paragraph, examples, notes, figures etc.) are described in "General text XML elements". In addition cross references (hyperlinks) can also be set up. The section "Module HTML bookmarks" describes how.

EXPRESS Description as part of the EXPRESS XML

The XML derived from the EXPRESS, e.g. arm.xml and mim.xml, comprises a set of XML elements representing every EXPRESS object. Each of these objects has a <description> element in which the description of the object is provided.

EXPRESS Type description

An example of a type description: 


TYPE work_order_approval_item = SELECT BASED ON approval_item WITH (Work_order);
END_TYPE; 
<type name="work_order_approval_item">
        
  <description>
    This select extends an approval_assignment to apply to Work_request 
    instances.
  </description>

  <select basedon="approval_item" selectitems="Work_order" />
  <graphic.element page="52" xcoord="538" ycoord="141" />
</type>

EXPRESS Entity and attribute description

An example of an entity description: 


ENTITY Activity; 
   name : STRING;
END_ENTITY;

<entity name="Activity">
        
  <description>
    An Activity is a specified operation usually doing some type of work to
    produce some identified output.  Status information specifying the level of
    completion of each activity may be specified.  The items that are affected
    by a spcific Activity, e.g. as input or output, may also be identified.  A
    description of the adopted solution procedure selected to carry out the
    action is also given.  
  </description>

<explicit name="name">
        
  <description>
    the label by which the activity is known.
  </description>

<builtintype type="STRING" />
</explicit>
</entity>

EXPRESS Formal proposition description

A formal proposition is a defined using an EXPRESS expression.

An example of the description of a Formal proposition: 


ENTITY Part
  SUBTYPE OF (Product) ; 
  WHERE
    WR1: SELF.product_type IN ['part', 'raw material'];
END_ENTITY;

    <entity name="Part" supertypes="Product">
        
        <description>
        A <b>Part</b> is a type of <b>Product</b> that is
        intended to be produced or employed in a production process. A Part
        may be either a single piece part, an assembly of arbitrary
        complexity, or a raw material.
        </description> 
        
      <where
        label="WR1"
        expression="SELF.product_type IN ['part', 'raw material']">
        
          <description>
          The <b>product_type</b> role shall be 'part', or 'raw
          material'.
          </description> 
        
      </where>
    </entity>

EXPRESS Informal proposition description

An informal proposition is a textual base constraint and is described in the XML by providing a WHERE rule with no EXPRESS expression.

An example of the description of a informal proposition: 


ENTITY Part
  SUBTYPE OF (Product) ; 
END_ENTITY;

    <entity name="Part" supertypes="Product">
        
        <description>
        A <b>Part</b> is a type of <b>Product</b> that is
        intended to be produced or employed in a production process. A Part
        may be either a single piece part, an assembly of arbitrary
        complexity, or a raw material.
        </description> 
        
      <where
        label="IR1"
        
          <description>
          The <b>product_type</b> role shall be 'part', or 'raw
          material'.
          </description> 
        
      </where>
    </entity>

EXPRESS Description through an external file

An alternative approach to documenting the EXPRESS in the XML derived from the EXPRESS is to provide the documentation in an external XML file. The XSL stylesheets will then combine the descriptions with the EXPRESS. If there are both forms of descriptions for any EXPRESS objects, then both will be displayed.

Identifying the description file

The external description file must be identified in the XML derived from the EXPRESS. This is done through the description.file parameter in the express element.

The example below shows the association of the arm_descriptions.xml file with the arm.xml file for the work_order. The arm_descriptions.xml file must be stored in the same directory as the arm.xml file. 

    <?xml version="1.0"?>
    <?xml-stylesheet 
        type="text/xsl" 
        href="../../../xsl/express.xsl"?> 
    <?DOCTYPE 
      express SYSTEM "../../../dtd/express.dtd"?>

    <express 
      language_version="2" 
      rcs.date="$Date: 2002/03/18 10:36:09 $"
      rcs.revision="$Revision: 1.6 $" 
      description.file="arm_descriptions.xml">
    <application 
      name="GraphicalEXPRESS" 
      owner="Eurostep" url="www.eurostep.com" 
      version="1.3" source="plcs_arm_ge13v54vis.vsd" />
      <schema name="Work_order_arm">
        ......
      </schema>
    </express>

External description file format

The format of the external description file is according to the following DTD: 
<!ENTITY % desctext SYSTEM "text.ent">
%desctext;
<!ELEMENT ext_descriptions (ext_description*)>
<!ATTLIST ext_descriptions
	module_directory CDATA #REQUIRED
	schema_file CDATA #REQUIRED
>

<!ENTITY % description.content "%text-elts; | %block-elts; | %single-block-elts; ">
<!ELEMENT ext_description (%description.content;)*>
<!ATTLIST ext_description
	linkend CDATA #REQUIRED
>

Each EXPRESS object should therefore have an <ext_description> element that provides its description.

The linkend attribute identifies the EXPRESS object for which the description applies. The syntax of the linkend string is described in "XML referencing".

The XML elements that can be used in the description text - as defined by description.content (e.g. paragraph, examples, notes, figures etc.) are described in "General text XML elements". In addition cross references (hyperlinks) can also be set up. The section "Module HTML bookmarks" describes how.

An example of an an external description file provided for the arm.xml file for the work_order module is give below. The EXPRESS that is being described is also shown. 

SCHEMA Work_order_arm; 

USE FROM Activity_arm;
USE FROM Activity_directive;
USE FROM Activity_method_arm;
USE FROM Approval_arm;
USE FROM Contract_arm;
USE FROM Date_time_assignment_arm;
USE FROM Person_organization_assignment_arm;
USE FROM Work_request_arm;

TYPE work_order_approval_item = SELECT BASED ON approval_item WITH (Work_order);
END_TYPE; 

TYPE work_order_contract_items = SELECT BASED ON contract_item WITH (Work_order);
END_TYPE; 

TYPE work_order_date_or_date_time_item = SELECT BASED ON date_or_date_time_item WITH (Work_order);
END_TYPE; 

TYPE work_order_organisation_or_person_in_organisation_item = SELECT BASED ON organization_or_person_in_organization_item WITH (Work_order);
END_TYPE; 

ENTITY Work_order; 
   description : OPTIONAL STRING;
   directed_activity : Directed_activity;
   in_response_to : SET[1:?] OF Work_request;
   name : STRING;
  DERIVE
    choosen_method : Directed_activity : Activity_directive_arm.Directed_activity.chosen_method;
    status : STRING : Directed_activity.status;
END_ENTITY;

END_SCHEMA;
