Introduction

This section describes the use of normative references, terms, definitions and abbreviations.
Normative reference
A reference to another standard
Term
A term used in this module. The term may be defined in another standard, in which case that standard must be identified as a normative reference
Definition
A term defined in this module
Abbreviation
An abbreviation for a term

Normative References

The approach to normative references in the module repository is to automate the inclusion of the references as far as possible.

This is achieved by:

  • providing a set of predefined normative references in the XML file: data/basic/normrefs.xml that can be referenced in the module documentation using <normref.inc>;
  • specifying a set of default normative references that get included in every module documentation by the XSL. These are defined in: data/basic/normrefs_default.xml;
  • the XSL style sheets automatically including all normative references to any module referenced by a USE FROM in the module ARM or MIM. All integrated resources referenced by a USE FROM in the MIM are also output for information. The normative reference will need to be added.
If any additional normative references are required, then they should either be added to data/basic/normrefs.xml and referenced in the module documentation using <normref.inc> or identified in the module documentation (module.xml) using the element <normref>.

As well as identifying that a normative reference issued by the module, the <normref.inc> element will also enable the identification of the terms in the normative reference that are used in the module. This is done through the <term.ref> element.

Defining a normative reference

A normative reference is either defined in data/basic/normrefs.xml or the module documentation module.xml.

If defined in data/basic/normrefs.xml then the top element is: 

  <!ELEMENT normref.list (normref+)>

If defined in module.xml then the top element is: 

  <!ELEMENT normrefs (normref*, normref.inc*)>

normref

A normative reference is defined by the XML element: 
<!ELEMENT  normref      (stdref, term*)>
<!ATTLIST  normref
             url        CDATA   #IMPLIED
             id         ID      #REQUIRED
             docbook-atts
                        CDATA   #FIXED "url #DEFAULT"
             docbook    NMTOKEN #FIXED "bibliomixed" >
where:
  • stdref provides bibliographic information about the normative reference.
  • term lists the terms defined in the normative reference and are used in this module.
  • id is the identifier given to the normative reference. This will be used by <normref.inc> to when including the normative reference in a module.

stdref

A definition of the standard reference, defined by: 
<!ELEMENT  stdref       (%bib-elts;)* >

<!ENTITY % bib-elts    "
  #PCDATA
  | orgname
  | pubdate
  | stdnumber
  | subtitle
  | stdtitle
  | ulink">
Where bib-elts are bibliographic elements.

The organization name is defined by: 

<!ELEMENT  orgname      (#PCDATA) >
<!ATTLIST  orgname
             docbook    NMTOKEN   #FIXED "orgname" >

The publication date is defined by: 

<!ELEMENT  pubdate      (#PCDATA)  >
<!ATTLIST  pubdate
             docbook    NMTOKEN   #FIXED "pubdate">

The standard number is defined by: 

<!ELEMENT  stdnumber  (#PCDATA) >
<!ATTLIST  stdnumber
             docbook    NMTOKEN  #FIXED "stdnumber">

The subtitle of the standard is defined by: 

<!ELEMENT  subtitle     (#PCDATA)  >
<!ATTLIST  subtitle
             docbook    NMTOKEN   #FIXED "subtitle">

The title of standard is defined by: 

<!ELEMENT  stdtitle     (#PCDATA) >
<!ATTLIST  stdtitle
             docbook    NMTOKEN   #FIXED "title">

A link to URL where the standard is held is defined by: 

<!ELEMENT  ulink        (#PCDATA) >
<!ATTLIST  ulink
             docbook    NMTOKEN  #FIXED "ulink"
             url        CDATA    #REQUIRED>

Example

An example of a normative reference is: 

   <normref id="ref10303-501.2000">
    <stdref>
      <orgname>ISO</orgname>
      <stdnumber>10303-501:2000</stdnumber>
      <stdtitle>
        Industrial automation systems and integration - Product data
        representation and exchange
      </stdtitle>
      <subtitle>
        - Part 501: Application interpreted construct: Edge based wireframe
      </subtitle> 
    </stdref>
  </normref>

A complete example is given in data/modules/example/module.xml. The results of the XSL stylesheet being applied are: data/modules/example/sys/2_refs.htm.

Terms

Terms are defined in standards and are identified in two ways.
  • Either terms may be defined as part of a module, in which case they form part of a definition.
  • Or terms are defined in some other a standard. If the standard is not a module in the repository, then the standard needs to be identified as a normative reference and the terms in the normative reference identified.
Hence part of the definition of a normative reference is the identification of the terms that have been defined in that standard. Once the term has been identified, it can be included as part of a normative reference in the module by using the <normref.inc> and <term.ref> elements.

The term is defined by the element: 

  <!ELEMENT  term         (#PCDATA)>                                 
  <!ATTLIST  term
             id         ID                                 #IMPLIED
             linkend    IDREF                              #IMPLIED
             url        CDATA                              #IMPLIED>
where <id> is the identifier by which the term will be referenced in <term.ref> element and hence included in the module documentation.

Example

An example of a term defined in a normative reference is: 

 <normref id="ref10303-1.1994">
   <stdref>
     <orgname>ISO</orgname> 
     <stdnumber>10303-1:1994</stdnumber>
     <stdtitle>
       Industrial automation systems and integration - Product data
       representation and exchange
     </stdtitle>
     <subtitle> - Part 1: Overview and fundamental principles</subtitle>
   </stdref>
   <term id="term1">application</term>
 </normref>

A complete example is given in data/modules/example/module.xml. The results of the XSL stylesheet being applied are: data/modules/example/sys/2_refs.htm.

Including a normative reference in a module

A normative reference can be included in the module documentation by the element: 
<!ELEMENT  normref.inc (term.ref*)>
<!ATTLIST  normref.inc normref NMTOKEN #IMPLIED>
<!ATTLIST  normref.inc module.name NMTOKEN #IMPLIED>
<!ATTLIST  normref.inc resource.name NMTOKEN #IMPLIED>
Where:
  • normref is the id of a normative reference to be included. The normative reference should be defined in data/basic/normrefs.xml or module.xml using normref.
  • module.name is the name of a module to be included as a normative reference.
  • resource.name is the name of an integrated resource to be included as a normative reference.

Example

An example of a normative reference being included in a module document is shown below. The normative reference is detailed in data/basic/normrefs.xml 

      <normref.inc normref="ref10303-501.2000"/>

A complete example is given in data/modules/example/module.xml. The results of the XSL stylesheet being applied are: data/modules/example/sys/3_defs.htm.

Identifying terms from normative reference for inclusion in a module

If any term is to be referenced in the module, then it must be specified as part of the normative reference that is being included in the module. This done using the element: 
  <!ELEMENT  term.ref     EMPTY>
  <!ATTLIST  term.ref     linkend            NMTOKEN         #IMPLIED>
where <linkend> is the id attribute of the <term>.

Example

An example of terms form a normative reference being included in a module document is shown below. The normative reference is detailed in data/basic/normrefs.xml 

      <normref.inc normref="ref10303-1.1994">
        <term.ref linkend="term1"/> <!-- application -->
      </normref.inc>

A complete example is given in data/modules/example/module.xml. The results of the XSL stylesheet being applied are: data/modules/example/sys/3_defs.htm. The results of the XSL stylesheet being applied are: data/modules/example/sys/3_defs.htm.

Identifying terms from another module for inclusion in a module

If a term defined in another module, is to be referenced in the module, then the element <normref.inc> must be used with the attribute <module.name> indicating the module where the term is defined.

Example

To include the module "example" as a normative reference and the use the term "test term" which is defined in the module "example": 

        <normref.inc module.name="example">
          <term.ref linkend="ttt"/>
        </normref.inc>

      ....

        <definition>
          <term id="ttt">test term</term>
          <def>The definition of the term test term.</def>
        </definition>

A complete example is given in data/modules/example/module.xml. The results of the XSL stylesheet being applied are: data/modules/example/sys/3_defs.htm.

Definitions

Each module may provide new definitions. These are defined using the element: 
<!ELEMENT  definition   (term, def)                                  >
<!ATTLIST  definition
             id         ID                                 #IMPLIED  >

<!ELEMENT  def          (%text-elts;)*                               >
Where term is the <term> being defined and <def> is the definition of the term.

Abbreviations

Abbreviations are abbreviations or acronyms for terms. Like normative references, abbreviations are defined in a common file in the repository for reuse. data/basic/abbreviations.xml and a set of abbreviations are defined as default in: data/basic/abbreviations_default.xml
