STEPmod Sourceforge project

The module development is a Sourceforge project: http://stepmod.sourceforge.net/

The Project Manager on Sourceforge is Josh Lubell (lubell@cme.nist.gov)

Using XML and Internet Explorer

In order to view the XML with XSL stylesheets dynamically applied, it is suggested that Internet Explorer version 6 be used.

Join email exploder

In order to participate in the discussions on the development of the XML repository and the modules themselves, join the STEPmod email exploder. http://lists.sourceforge.net/lists/listinfo/stepmod-devel

Become a developer

In order to become a module developer first create a Sourceforge account, then contact the Josh Lubell (lubell@cme.nist.gov) to be added as a project developer.

You will need CVS and a secure shell to be able to participate.

To create a Sourceforge account, go to http://sourceforge.net/projects/stepmod and use the menu option on the left "New User via SSL"

Getting started with CVS

Compiled by: Rob Bodington (rob.bodington@eurostep.com)

Here are some additional notes on how to get CVS working with SourceForge.

This details the software I am using - there are other ways of doing this.

This help was based on Josh's help page at http://ats.nist.gov/stepmod/cvswin.html.

  1. Install WinCVS - Can be down loaded from: http://www.wincvs.org/
  2. Install the SSH software PuTTY - You need putty.exe, plink.exe, pageant.exe, puttygen.exe. The putty help pages would be a useful download as well. These can be down loaded from http://www.chiark.greenend.org.uk/~sgtatham/putty/. Useful help pages: http://sourceforge.net/docman/display_doc.php?docid=6841&group_id=1
  3. Set up and test your SSH to connect to SourceForge. First generate and post your SSH key to SourceForge. Details on how to do this are at http://sourceforge.net/docman/display_doc.php?docid=761&group_id=1. Note - you will have to wait 6 hours before your key has been circulated.
  4. Create and save a putty session that logs on to SourceForge using SSH. On the session window: 
         Host Name: cvs.sourceforge.net
         SSH selected

On the SSH window (the SSH window is obtained by clicking Auth on the left side of the Putty window): 

   SSH 1 selected

Specify your authentication key.

  1. Set up PLINK and CVS environment variables 
         set CVS_RSH=\path\to\plink.exe (that means you have to wrire:
                                        set CVS_RSH=C:\……..i.e. the path to plink.exe in your computer)
         set PLINK_PROTOCOL=ssh
  1. Run pageant adding your key  (that should be done any time you will switch on or reboot your computer, including the adding of your key)
  2. Configure WinCVS
    Admin->Preferences->General
    CVSROOT box
    :ext:putty_session_name:/cvsroot/stepmod
    where putty_session_name is the name of the putty session you just created.
    Admin->Preferences->Ports
    Alternative rsh name
    C:\………        (i.e. the path to plink.exe in your computer)
  3. Check out the repository - use the Admin->Command line
         cvs -d:ext:username@putty_session_name:/cvsroot/stepmod co stepmod

Creating a release

To make a release, you should create a tag for the release in CVS, checkout the tagged reflease, zip it and upload it as a release to Sourceforge.

For a Guide to the SourceForge.net File Release System (FRS) see:
http://sourceforge.net/docman/display_doc.php?docid=6445&group_id=1

The process for making a release is:

  1. Update release notes stepmod/releasenotes/ and stepmod/help/releases.htm
  2. Make sure everything is checked into CVS on SourceForge;
  3. Tag a release in CVS
    cvs tag stepmod_2002-01-03 .
    where stepmod_2002-01-03 is the tag for the release.
  4. Export the release from CVS / SourceForge
    cvs -d :ext:stepmod:/cvsroot/stepmod export -r stepmod_2002-03-04 stepmod
    better to check out the release as it can then be safely updated
    cvs -P -d:ext:robbod@cvs.sourceforge.net:/cvsroot/stepmod co stepmod
  5. Zip release stepmod_2002-01-03
  6. Upload zip file to file release system on SourceForge
    • FTP to upload.sourceforge.net
    • Login as "anonymous"
    • Use your e-mail address as the password for this login
    • Set your client to binary mode ("bin" on command-line clients)
    • Change your current directory to /incoming ("cd /incoming")
    • Upload the desired files for the release ("put filename")
  7. Create a new release on SourceForge
