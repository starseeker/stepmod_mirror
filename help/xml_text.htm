Text XML elements

This section describes the XML elements can be used when documenting a module. The elements are defined in DTDs.
  • Paragraph
  • List
  • Screen
  • Example
  • Notes
  • Figures
  • Images

Paragraph <p>

Paragraphs are defined by the XML element (these are the same as the HTML elements of the same name): 
  <!ELEMENT p (%text-elts;)*>
E.g. 
  <p>A new paragraph.</p>

List <ul|ol>

Lists are defined by the XML elements (these are the same as the HTML elements of the same name): 
  <!ELEMENT ul (li+)>
  <!ELEMENT ol (li+)>
  <!ATTLIST ol
		type (1 | A | a | I | i) "1"
 		start CDATA #IMPLIED>

  <!ELEMENT li (%text-elts;)*>
where <ul> is an unordered list, <ol> is an ordered list and <li> is a list item.

E.g. The following XML: 

  <ul>
     <li>First item.</li>
     <li>Second item.</li>
  </ul>
results in:
  • First item
  • Second item

Images <imgfile|img>

Images can either be inserted into the current text by using <img> or as a link which will display the image in a new browser window using <imgfile>

Linked image <imgfile>

The element <imgfile> is defined by: 
  <!ELEMENT imgfile EMPTY>
  <!ATTLIST imgfile file CDATA #REQUIRED>
<imgfile> is used to link to an image that is to be displayed in a new browser window. The linked image that will be displayed is defined a separate file comprising of a <imgfile.content> element which defines the actual image to be displayed. The element <imgfile.content> is defined by: 
  <!ELEMENT imgfile.content (img)>
  <!ATTLIST imgfile.content 
          module CDATA #REQUIRED
          title  CDATA #IMPLIED>
where:
  • module is the name of the module in which the image is defined.
  • title is the title to be displayed with the image.
An example of where this is used is listing the pages of an EXPRESS-G diagram. Each page will be a different image and identified by a <imgfile>. The diagram being pointed to are defined in separate files each comprising of a <imgfile.content> element which defines the actual image to be displayed. E.g.

module.xml 
      <arm>
        <express-g>
          <imgfile file="armdiag1.xml"/>
          <imgfile file="armdiag2.xml"/>
        </express-g>
      </arm>

armdiag1.xml 
      <imgfile.content
        module="ModuleName"
        title="the title">
        <img src="arm.gif"/>
      </imgfile.content>

In lined image <img>

An image to be inserted into the current page is defined by: 
  <!ELEMENT img (img.area*)>
  <!ATTLIST img
          src CDATA #REQUIRED
          alt CDATA #IMPLIED>
where:
  • src is the URL of the graphic to be displayed.
  • alt is a string of alternative text that appears when the graphic is not displayed.

Mapped image <img.area>

An area of the image defined as clickable image map. It is defined by: 
  <!ELEMENT img.area  EMPTY>
  <!ATTLIST img.area
	shape CDATA #REQUIRED
	coords CDATA #REQUIRED
        href CDATA #REQUIRED>
where:
  • shape is the shape of the clickable area.
  • coords comma separated coordinates that define the clickable area of the image.
  • href is the URL for the area.
The <img.area> is equivalent to the HTML element <area>

An image map can be created from an HTML page using the utility imagemap2xml.js

An example of imgfile.content, img and img.area is:

armdiag.xml 
<?xml version="1.0"?>
<?xml-stylesheet type="text/xsl" href="../../../xsl/imgfile.xsl"?>
<!DOCTYPE imgfile.content SYSTEM "../../../dtd/text.ent">
<imgfile.content
  module="person"
  title="the title">
  <img
    src="arm.gif">
    <img.area
      shape="RECT"
      coords="12,120,220,173"
      href="../../pdm/product_version/armdiag.html#product_version"/>
    <img.area
      shape="RECT"
      coords="11,192,277,245"
      href="../../pdm/product_view_definition_structure/armdiag.html#view_definition_relationship"/>
  </img>
</imgfile.content>
